# Making Fiscal Policy Legible

India's Union Budget 2024 allocated ₹47.65 lakh crore across 46 ministries. The official documentation runs to 4,700 pages. Most people — including most journalists — engage with it through a handful of headline numbers.

This project asked: what would it look like if someone actually tried to make the whole thing visible?

## Data Architecture

The source data comes from the Union Budget documents, supplemented by RBI databases and Ministry-level detailed demands for grants. Wrangling it into a usable shape took three weeks.

The core challenge: India's budget uses a functional classification system that doesn't map cleanly to economic sectors. "Defence" includes civilian employees of defence establishments. "Education" is split across Union and State lists. "Infrastructure" doesn't exist as a budget category at all.

I created a secondary classification layer: 12 economic themes mapped from the 46 ministerial heads, with documented mapping rules. This is the controversial design decision — any mapping introduces judgment. The mapping rules are documented and explorable in the visualization.

## Visual Language

The primary visualization is a modified Sankey diagram showing budget flow from revenue sources → consolidated fund → expenditure heads → economic themes.

Key decisions:

- **Color**: Expenditure vs. capital vs. revenue distinctions use a three-value sequential scale (not categorical) — the distinction is about type of spending, not about the ministries themselves
- **Scale**: All figures displayed as percentage of GDP, not absolute rupee values — the absolute numbers are incomprehensible, the percentages are comparable across years
- **Comparison**: 2020–2024 comparison is always available as a parallel view — no visualization of budget data should exist without historical context

> "Showing one year of budget data is propaganda. Showing five years is analysis."

## Technical Build

D3.js v7 for all visualization components. Custom Sankey layout implementation (the standard d3-sankey didn't support the nested hierarchy I needed). React wrapper for interactivity — hover states, year selection, drill-down navigation.

The most interesting technical challenge: the diagram had to be fully accessible. All visual relationships encoded in D3 SVG have ARIA descriptions generated programmatically from the underlying data. Keyboard navigation traverses the flow graph in topological order.

## Limitations and Honesty

The project page includes a prominent "What this doesn't show" section:
- Off-budget financing (still partially opaque even to experts)
- State-level spending (Union Budget covers only the centre)
- The secondary mapping layer's inherent subjectivity

Dataviz that hides its limitations is deceptive. Dataviz that foregrounds them earns trust.
